# LIFE-Mind 行为与动画状态机 v1

## 目标

桌宠的动作必须能回答三个问题：她现在是什么状态、为什么做这个动作、动作结束后去哪里。
语言模型只能提供语言理解和自然措辞，最终情绪、活动、冷却和动作切换由本地确定性状态机决定。

## 三层状态

1. 心智层：`energy / mood / trust`，保存在 SQLite，重启后延续。
2. 行为层：`dominant_emotion / current_activity / cause / activity_until / cooldowns`。
3. 动画层：`enter -> loop -> exit`。反应动作可短暂打断活动，但结束后必须回到可解释状态。

## 用户语言映射

| 用户表达 | 情绪 | 动作 | 状态影响 |
|---|---|---|---|
| 问候、回来啦 | 开心、亲近 | 招手/认真看向用户 | 心情小幅上升 |
| 提问、为什么、怎么 | 好奇 | 歪头思考 | 不强改心情 |
| 震惊、居然、感叹 | 轻震惊 | 睁大眼后恢复 | 短反应，不长期残留 |
| 谢谢、喜欢、夸奖 | 害羞开心 | 微笑、轻轻雀跃 | 心情和信任上升 |
| 辛苦、休息、没关系 | 放松、被理解 | 松一口气 | 心情和信任上升 |
| 明确任务、继续工作 | 专注 | 进入电脑工作 | 精力缓慢下降 |
| 具体纠错、不对、重来 | 认真反思 | 低头思考后抬头 | 心情轻微下降，不自我否定 |
| 人格羞辱、恶意驱赶 | 低落并收敛表达 | 安静、减少主动动作 | 心情明显下降，信任轻微下降 |
| 道别、晚安 | 温柔平静 | 招手后回到独处状态 | 不制造内疚或挽留 |

## 独处行为

- 精力 `< 0.28`：休息权重最高。
- 心情 `< 0.38`：更可能安静画画、发呆或休息，不播放夸张开心动作。
- 精力中等：画画、短休息、浇花和待机。
- 心情 `> 0.80` 且精力充足：轻轻哼唱、浇花、画画、环顾。
- 普通稳定状态：低打扰待机、画画、浇花、偶尔整理自己的小项目。

独处动作不自动弹语言符号。启动后至少安静 18 秒，用户说话后至少安静 14 秒；同一活动有
55–150 秒冷却，活动持续 8–80 秒。随机数只在多个合理候选中选择，不能突破精力和情绪约束。

## 动画切换

- 站立到坐姿活动：`idle -> sit_down -> activity_loop`。
- 坐姿活动结束：`activity_loop -> stand_up -> idle`。
- 用户在坐姿活动中发言：先经过单人物淡出/淡入过门，再完整起身回应，不瞬移。
- 情绪反应：`neutral -> reaction_enter/loop -> settle -> previous_or_idle`。
- 每组正式动作统一 48 帧；循环活动先走到设计好的收势点再切换，跨动作通过六帧单人物淡出/淡入过门衔接，不把两个人形透明叠在一起。
- 站立与坐姿之间完整播放坐下/起立；两个坐姿活动互换时保持坐姿，不再多余地站起再坐下；反应动作至少完整播放一次并在末帧收束。
- 站姿固定 320px、坐姿（含睡眠）固定 278px，头部横向锚点误差不超过 2px、脚底基线误差为 0px。
- 不允许两个差异很大的姿势直接切换，也不通过模糊缩放伪造流畅。

## 优先级

`用户直接互动 > 必要状态恢复 > 用户手动活动 > 当前活动完成 > 独处自主选择 > 装饰性微动作`

高优先级可以中断低优先级，但必须播放退出或收束过渡。普通独处选择不能打断用户对话。
