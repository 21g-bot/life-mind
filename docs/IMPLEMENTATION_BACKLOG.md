# GitHub 实施任务

本页是公开路线图的工程分解。任务编号稳定，可把任一小节原样复制为 GitHub Issue。优先级：
P0 阻塞下一个可用版本，P1 构成完整体验，P2 在核心稳定后扩展。规模 S/M/L 只表示相对工作量。

## Milestone 0.2：可安装、好玩、宿主真正解耦

### HOST-01｜P0｜M｜让参考宿主只消费 PresentationIntent

- 依赖：FND-02。
- 交付：Tkinter 宿主不再读取完整决策 trace，只读取 `PresentationIntent`；开发者调试器走独立审计通道。
- 验收：普通宿主对象拿不到关系分项、候选分数或成长门槛；缺失 clip 回退 idle；现有动作切换测试全部通过。
- 非目标：本任务不更换 Tkinter，也不增加 Live2D。

### PKG-01｜P0｜L｜Life Package 本地加载器和 lockfile

- 依赖：FND-01。
- 交付：目录/ZIP 校验、兼容版本、素材许可证、SHA-256、解包目录限制、实例种子和 `life-package.lock.json`。
- 验收：拒绝目录穿越、重复文件、超限包、未知版本和缺失署名；离线安装/卸载可重复；不执行包内代码。
- 非目标：没有远程市场，没有自动下载。

### HOST-02｜P0｜M｜气泡预算、优先级、去重和安静时段

- 依赖：HOST-01。
- 交付：气泡调度器，支持 `priority`、冷却、相似文本去重、勿扰和用户交互抢占。
- 验收：普通主动气泡不超过配置预算；权限/错误事件可抢占；同一句不会短时间重复；固定时钟测试确定。

### LIFE-01｜P1｜M｜第一个“任务外生活”循环

- 依赖：HOST-01。
- 交付：画画或照料产生可收藏作品/纪念物；过程消耗时间和精力，可成功也可中止。
- 验收：不用 LLM 可完成；作品来源能追溯到自主事件；不会因为用户离开而惩罚或制造内疚；普通 UI 不显示内部数值。

### ITEM-01｜P1｜L｜物品、作品与房间数据包

- 依赖：PKG-01、LIFE-01。
- 交付：只含数据和素材的 Item/Room Package，支持本地校验、许可声明和 Life Package 兼容范围。
- 验收：第三方内容不需要改核心代码；素材缺失有安全回退；背包数值不能直接改人格或关系。

### SPACE-01｜P1｜L｜只读桌面空间适配器

- 依赖：HOST-01。
- 交付：Windows 任务栏、工作区、多屏幕、前台窗口矩形和安全坐点快照；默认关闭详细窗口标题采集。
- 验收：角色可避开屏幕外和关键操作区；不会自行巡游；快照默认不记忆、不进 LLM；关闭权限后立即停止采样。

### DIST-01｜P0｜L｜Windows 安装、升级和卸载

- 依赖：PKG-01。
- 交付：签名策略说明、构建脚本、安装包、数据保留选项、卸载和损坏安装诊断。
- 验收：全新 Windows 用户无需 Python；卸载不误删用户导出的包；Release 构建可从固定提交重现。

### QA-01｜P0｜M｜8 小时桌面与资源预算闸门

- 依赖：HOST-02、DIST-01。
- 交付：CPU、内存、线程、句柄、动画帧延迟和数据库错误统计报告。
- 验收：达到 `MVP_ACCEPTANCE.md` 预算；异常退出可恢复；报告脱敏并可附到 Release。

## Milestone 0.3：安全能力运行时

### CAP-00｜P0｜M｜威胁模型与隔离技术选型

- 依赖：FND-01。
- 交付：资产、攻击者、信任边界、Windows 隔离限制和候选运行时实验；明确 Python 子进程为何不自动等于沙箱。
- 验收：覆盖文件、网络、凭据、IPC、ZIP、更新链和拒绝服务；决策记录进入 `docs/decisions/`。

### CAP-01｜P0｜L｜能力生命周期与用户授权存储

- 依赖：CAP-00。
- 交付：install/enable/start/stop/disable/uninstall 状态机；授权按版本、权限和数据范围保存。
- 验收：新增权限必须重新授权；撤权立即停止；崩溃不影响桌宠；每次状态变化有审计事件。

### CAP-02｜P0｜L｜真正的隔离执行宿主

- 依赖：CAP-00、CAP-01。
- 交付：无任意 shell、Node/Python 导入、直接文件系统或原始网络；只注入最小能力对象。
- 验收：恶意测试包无法读环境变量、用户目录或打开任意 socket；超时/崩溃被终止；宿主仍可运行。

### CAP-03｜P0｜L｜宿主代理网络与 SSRF 防护

- 依赖：CAP-02。
- 交付：仅 HTTPS、精确主机白名单、DNS 重绑定/私网/重定向检查、大小和时间限制。
- 验收：localhost、链路本地、私网、非声明域名和跨域重定向均被拒绝并审计。

### CAP-04｜P1｜M｜配额、独立存储、撤权和审计

- 依赖：CAP-01、CAP-02。
- 交付：每能力的调用/通知/存储配额，虚拟时钟测试，一键导出审计摘要。
- 验收：配额不能通过重启绕过；插件只能访问自身命名空间；日志不含敏感正文。

### CAP-05｜P1｜L｜MCP 只读首批适配器

- 依赖：CAP-03、CAP-04。
- 交付：先支持状态类、无需凭据的只读工具；结果统一转 Experience Protocol。
- 验收：工具无法直接写 Mind Core；每次调用显示来源、权限、用途和失败原因；模型不能自行扩大授权。

### SUPPLY-01｜P0｜M｜包溯源、哈希和安全更新

- 依赖：PKG-01、CAP-01。
- 交付：来源仓库、固定提交/标签、SHA-256、发布者和权限差异报告。
- 验收：包 id/版本/哈希不匹配拒绝；增加权限、域名或换发布者禁止静默升级。

## Milestone 0.4：统一经验与长期记忆

### EXP-01｜P0｜M｜所有外部输入统一走 Experience Ingress

- 依赖：HOST-01、CAP-01。
- 交付：用户输入、桌面状态、能力和自主活动共用验证、去重、来源与权限检查。
- 验收：重复 eventId 幂等；未知来源拒绝；观察事实和角色解释分字段保存。

### MEM-01｜P0｜L｜分层记忆保留与检索预算

- 依赖：EXP-01。
- 交付：原始、情景、关系、语义和叙事层；每层保留期、用途和检索上限。
- 验收：模型上下文只拿到允许用途；不确定信息不升级为事实；长期数据规模测试有性能报告。

### MEM-02｜P0｜L｜撤权和删除的派生污染追踪

- 依赖：MEM-01。
- 交付：`derived_from` 图、级联删除/降级、索引重建和验证报告。
- 验收：删除源事件后，单独依赖它的摘要和信念不可再检索；多来源结论降低置信度并保留剩余证据。

### DATA-01｜P1｜M｜迁移、备份、恢复与损坏演练

- 依赖：MEM-01。
- 基础交付：架构版本与迁移记录、失败回滚、SQLite 原子快照、七份保留、损坏副本隔离、
  最近有效备份自动恢复、手动恢复入口和脱敏 `doctor` 已实现；
- 已完成：10,000 条事件的确定性重放、备份、主动损坏、自动恢复、耗时、数据库体积和 RSS
  预算闸门；常驻内存只保留最近 128 条完整轨迹，完整审计记录仍保存在 SQLite；
- 后续：面向普通用户的恢复预览窗口，以及发布候选版的真实断电演练；
- 验收：失败迁移可回滚；备份不含模型密钥；旧版公开数据夹具可升级；损坏的最新备份会被
  跳过，当前数据库在手动恢复前会保留到隔离目录；被强制终止后，已提交事件保留、未提交
  WAL 事务回滚；10,000 条验收必须满足仓库定义的资源预算。

## Milestone 0.5：可信成长

### SELF-01｜P0｜L｜Core / Current / Narrated Self 三层模型

- 依赖：MEM-01。
- 交付：稳定范围、证据状态和单次表达快照分开持久化；只有程序门槛能更新前两层。
- 验收：模型文本无法直接写 Core/Current；每次叙述可追溯到模型、事件和允许记忆。

### DRIFT-01｜P0｜L｜人格漂移检测与表达重生成

- 依赖：SELF-01。
- 交付：价值冲突、身份矛盾、关系越权和无证据自述检测；拒绝、降级或重生成策略。
- 验收：对抗样本不能让角色突然改价值底线、声称不存在经历或把一次情绪写成永久人格。

### ARC-01｜P1｜L｜成长弧模板和回归夹具

- 依赖：SELF-01、MEM-02。
- 交付：只描述核心矛盾、允许转折和证据门槛的 Arc Package；禁止预写结局。
- 验收：同一出生包可因事件走向不同但合理的阶段；推进需要选择、代价、跨情境重复和一致反思。

### UX-01｜P0｜M｜真实日/周节奏与行为盲测

- 依赖：DRIFT-01、ARC-01、QA-01。
- 交付：至少 5 名匿名体验者、脱敏卡片、预注册判定规则和结果汇总。
- 验收：至少 4/5 能仅根据行为识别人物弧；失败时不靠改台词掩盖行为问题。

- 验收：GUI 生成包与手写包格式完全相同；不能设置脚本化结局或越过安全范围。

### INDEX-01｜P2｜L｜可自托管包索引

- 依赖：SUPPLY-01、CLI-01、完成安全审计。
- 交付：静态索引格式、来源/许可证/权限展示和本地镜像；主程序不绑定单一服务。
- 验收：用户可添加或移除索引；索引不能覆盖 lockfile；下架不会删除本地合法内容。

## 所有 Issue 的统一 Definition of Done

1. 有自动测试，并写明仍需哪些人工验证；
2. 不提交私人角色素材、运行数据库、真实聊天、密钥或本机路径；
3. 新公开字段有 Schema、兼容策略、示例和迁移说明；
4. 新能力说明观察、记忆、使用、行动和分享边界；
5. UI 功能通过黑箱审查，不泄露内部关系分项或候选分数；
6. 不让 LLM 或第三方包直接修改人格、权限、关系和成长阶段；
7. 运行完整测试、协议校验、文档链接检查和公开发布检查；
8. 使用第三方代码或素材时记录来源、许可证、版本和 NOTICE 要求。
