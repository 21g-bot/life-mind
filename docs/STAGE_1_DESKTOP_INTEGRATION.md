# 阶段 1：桌宠心智集成

## 已接通的数据流

`用户对话 / 指定活动 / 自主活动 -> MindEventBridge -> PersistentMindRuntime -> 统一仲裁 -> 动画与表达`

- `life_mind/integration.py` 把实际输入转换成结构化事件，并把心智动作映射到像素动作名。
- `life_mind/persistence.py` 保存事件和完整轨迹，并用固定 seed 在启动时回放。
- `life_mind/mind.py` 负责旧状态迁移、统一状态同步、反思证据门禁和本地 AI 表达适配。
- `life_mind/apps/desktop_pet.py` 只执行仲裁结果，并在显式开发者模式提供可视化心智调试器。

## SQLite 表

- `mind_runtime_meta`：固定 seed、初始迁移状态和最近状态元数据。
- `mind_events_v2`：不可变事件、事件 ID、完整仲裁轨迹和写入时间。
- 原有 `state`、`events`、`memories` 表继续兼容旧安装；首次启动后由统一心智状态同步。

同一个 `event_id` 重复写入不会产生第二次状态变化。数据库重启后按顺序回放全部有效事件，
因此状态、关系、成长门槛和最后选择可以复现。损坏的单条事件会记录到调试器的回放异常区。

## 心智调试器

使用 `py -B run_pet.py --developer-mode` 启动，再选择“开发者：心智调试器…”后可查看：

1. 身体、需要、注意、情绪、稳定人格和当前 tick；
2. 用户、友善指导者、尖锐评论者三条独立关系；
3. 最近一次社会评价、全部候选、得分、拒绝原因和最终动作；
4. 四阶段成长证据与当前缺口；
5. 最近持久化事件的回放序列。

测试事件注入会先提示其将写入真实事件历史。正式验收应始终使用独立 `--db-path`，
避免污染用户的长期记忆。

普通启动的右键菜单不会出现状态摘要或心智调试器，个人房间也不会复用这里的内部快照。

## 视觉运行约束

运行时只接受声明 `style=refined-pixel-art` 且具有非空 `identity` 的像素动作库。本机可以
用 `LIFE_MIND_CHARACTER_IDENTITY` 锁定指定身份。私人动作库存在时优先使用；公开仓库没有
私人素材时自动生成中立演示角色。旧来源包、单个 GIF 和骨骼捕获没有启动回退路径。

界面验收可使用 `--ui-qa-lightweight`，它只加载正式像素库中的 `idle` 与 `blink`，并不
加载、复制或回退到任何旧素材。

## 当前发布缺口

- 实际跑满 8 小时的运行、内存与事件回放稳定性门槛。
