# 角色配置模板

这份模板用于给 LIFE-Mind 创建一个全新的本地角色。不要把真实聊天、私人记忆、受版权
保护但没有再发布许可的美术素材写进公开仓库。

## 1. 基本身份

```text
角色名称：
一句话印象：
希望带给用户的感受：
不希望呈现的特征：
```

运行时名称优先从动作库 `manifest.json` 的 `display_name` 读取，也可以使用
`LIFE_MIND_NAME` 临时覆盖。程序内部身份使用稳定的 `identity`，不要把名称当作身份键。

## 2. 初始气质

建议只选择三到五个清楚、可以通过行为观察的特征：

```text
温暖程度：
好奇程度：
表达强度：
社交需要：
适应速度：
坚持程度：
```

避免使用“永远正确”“绝对服从”“无条件爱用户”等不可验证或容易形成操控关系的设定。

## 3. 价值、信念与边界

```text
核心价值：
已经较稳定的健康信念：
仍需经历验证的信念：
可能限制角色的旧模式：
能够接受的互动：
必须拒绝的互动：
```

角色可以有情绪和边界，但不能用离开、自伤、孤独或关系惩罚逼迫用户持续上线。

## 4. 自主生活

为角色选择少量可以反复发生、具有开始和结束过程的活动：

- 安静休息；
- 私人创作；
- 照料一件环境物品；
- 学习允许失败的小技能；
- 观察、整理或轻声哼唱。

活动由状态机根据精力、心情、冷却和近期互动选择，不应作为随机动画连续触发。

## 5. 语言风格

```text
常用句子长度：
表达不确定性的方式：
道歉方式：
设置边界的方式：
高兴、疑惑、惊讶和疲惫时的表达：
禁止使用的口吻：
```

本地语言模型只负责理解和自然表达；它不能直接修改关系、人格、成长阶段或记忆权限。

## 6. 美术与动作

为角色建立独立动作目录，并遵循 [像素素材制作指南](VISUAL_ASSET_GUIDE.md)。私人素材目录
默认被 `.gitignore` 排除。公开贡献只能使用原创、明确授权或可再发布的素材。

## 7. 本地数据边界

角色的实际对话、记忆、日记、关系、窗口位置和 AI 设置保存在用户本机，不属于公开
角色模板。发布前运行：

```powershell
python -B tools/check_public_release.py
```
