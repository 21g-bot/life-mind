# 阶段 0：无界面心智模拟器

## 目标

在不依赖桌宠窗口和语言模型的情况下，验证角色的状态、关系、边界、行为选择和
人物成长具有确定性、可解释性与可回放证据。

语言模型只能在后续适配层解释输入和生成自然表达，不能直接修改这里的关系、人格、
权限或成长阶段。

## 已实现闭环

`结构化事件 -> 社会评价 -> 候选行为 -> 安全过滤 -> 固定种子仲裁 -> 状态变化 -> 记忆 -> 成长门槛 -> 调试轨迹`

主要代码：

- `life_mind/domain/models.py`：事件、身体、需要、注意、情绪、稳定人格、关系、成长和记忆契约。
- `life_mind/simulator.py`：事件评价、行为候选、安全仲裁、关系修复、成长阶段和回放报告。
- `simulations/demo_growth.json`：公共参考人物弧的 15 个固定事件。
- `tests/test_mind_simulator.py`：确定性、关系修复、非操控、成长门槛和 100 天人格稳定测试。

## 运行

运行一次人物弧：

```powershell
cd life-mind
py -B -m life_mind.simulator
```

用同一 seed 重放 30 次：

```powershell
py -B -m life_mind.simulator --seed 20260717 --repeat 30
```

输出每个事件的状态前后差异、候选动作、拒绝原因、选中动作、记忆和成长证据：

```powershell
py -B -m life_mind.simulator --json
```

也可以复制 `simulations/demo_growth.json`，修改事件后通过 `--scenario` 注入：

```powershell
py -B -m life_mind.simulator --scenario simulations\my-scenario.json --json
```

## 成长硬门槛

首条人物弧严格按顺序推进，每个事件最多前进一个阶段：

1. `WORK_DEPENDENT`：依赖工作证明价值。
2. `CONFLICT_ACCUMULATING`：至少两次疲劳工作，并出现一次旧模式觉察。
3. `FAILURE_CRISIS`：高疲劳下出现失败，并积累至少两次觉察证据。
4. `INDEPENDENT_VALUE`：至少三次主动选择，在至少两个不同情境重复，承担累计代价，且有一条与结构化证据一致的反思。

一条 AI 反思、一次夸奖或一个极端事件都不能直接改变成长阶段。

## 关系修复硬门槛

关系按人物独立存储。伤害后，一句“对不起”只被记为表达，不恢复安全感。完整修复需要：

1. 承认具体伤害；
2. 承担责任；
3. 提供补救；
4. 出现行为改变；
5. 经过时间验证。

五类证据齐备后，信任和安全感只会逐步回升，不会恢复成“从未受伤”。

## 桌宠集成状态

阶段 0 契约现已通过 `PersistentMindRuntime` 接入桌宠 SQLite：每个结构化事件和完整
仲裁轨迹都会持久化，重启后使用同一 seed 进行事件回放。对话、用户指定活动和自主
行为都通过 `MindEventBridge` 进入同一仲裁器，桌宠动画只执行选中的动作。

显式使用 `--developer-mode` 启动后，右键“开发者：心智调试器…”可以查看状态、三人
关系、候选得分、安全拒绝、成长证据和事件回放。普通模式不提供这些入口。语言模型仍然
只负责输入解释与自然表达，不能直接修改关系、稳定人格或成长阶段。

完整 MVP 尚余个人房间、记忆权限级联删除与导出、系统托盘/勿扰模式及 8 小时稳定性
门槛；这些功能必须继续复用本阶段事件契约。
