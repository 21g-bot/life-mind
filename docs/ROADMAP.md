# LIFE-Mind 路线图

路线图按用户可感知的价值推进，而不是同时铺开所有心理变量。仓库里已经存在部分后期研究
原型，但这不代表对应产品阶段已经交付。

本页描述产品顺序；带稳定编号、依赖和验收条件的工程拆分见
[GitHub 实施任务](IMPLEMENTATION_BACKLOG.md)。参考项目的吸收依据见
[研究与吸收矩阵](REFERENCE_PROJECT_STUDY.md)。

## MVP 1：先让她“活着”

目标是即使没有 LLM，桌宠仍然可爱、安静、可互动并拥有自己的日常节奏。

当前已有：

- Windows 透明像素 Sprite 参考宿主、拖拽、托盘、勿扰和气泡；
- 16 个公开演示动作和带冷却、持续时间、情绪条件的行为状态机；
- 工作、休息、玩耍、照料等活动，断开模型仍可运行；
- 黑箱个人房间与开发者调试入口。
- 宿主无关的 Renderer 端口和 `PresentationIntent` 表现压缩层。

进入稳定发行前仍需：

- 8 小时真实桌面稳定性和资源预算实测；
- 窗口边界、多屏幕和遮挡区域感知；
- 触摸反馈、物品、纪念品、作品收集和至少一个轻量小游戏；
- 面向普通用户的安装包、升级与卸载体验。

## MVP 2：再让她“会做事”

目标是让外部能力可安装、可撤销、可审计，而不是让心智直接调用任意代码。

当前只有：Capability Manifest 0.1、网络白名单结构、明确授权判定和公开示例。

仍需实现：

- 受信任插件加载器、独立进程沙箱和每插件独立存储；
- 调用配额、生命周期、超时、撤权和完整审计；
- SSRF 防护、出站网络代理和敏感内容过滤；
- MCP/工具适配器及“工作中/等待授权/完成”表现状态；
- 签名包、诊断命令和一键安装/停用流程。

## MVP 3：再让她“记得”

目标不是保存所有聊天，而是保存少而重要、带来源和用途边界的经历。

当前参考实现已有：SQLite 事件溯源、记忆来源与置信度、隐私级别、允许用途、纠正/删除/
导出、关系分项和确定性回放。Experience Protocol 0.1 提供统一输入边界。

仍需实现或加固：

- 不同宿主和能力对 Experience Protocol 的统一接入；
- 原始事件、情景、语义、关系和叙事记忆的分层保留策略；
- 撤权后的派生记忆污染追踪和跨索引删除证明；
- 已建立版本迁移、原子备份、损坏隔离、自动恢复和脱敏诊断；10,000 条长期数据的有界轨迹
  缓存、重放与恢复性能闸门已经通过，仍需发布候选版的真实断电演练。

## MVP 4：最后让她“成长”

只先证明一条完整人物弧：她原本认为只有工作才有价值，后来通过失败、私人兴趣、新选择
和关系修复，逐渐形成更独立的自我价值。

当前已有：四阶段确定性模拟、跨情境证据门槛、关系伤害与修复、成长回放、盲测材料生成。

仍需完成：

- Core Self / Current Self / Narrated Self 自动漂移检测；
- 真实日/周节奏下的长期运行验证，而不只依赖加速模拟；
- 体验者盲测证明用户能从行为看出变化；
- 角色作者可复用的人物弧模板、回归测试和调参工具。

## 平台化后续

四个 MVP 稳定后，才扩展 Live2D/Spine 等渲染器、跨平台宿主、Creator Studio 图形界面、
包索引或内容市场。第三方生态必须晚于权限、许可证和撤权机制，不能以便利为由绕过安全边界。

每个版本的实际验收以测试和发布记录为准。路线图中的“仍需”项目不是当前功能承诺。
