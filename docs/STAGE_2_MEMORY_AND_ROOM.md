# 阶段 2：记忆治理与个人房间

## 已实现

### 可撤销记忆链

- 每条长期记忆保存来源、隐私级别、允许用途、置信度、重要性、待复核状态和派生来源。
- `memory_search_index` 是独立检索索引；删除或降级后会同步移除，不会继续被召回。
- `memory_dependencies` 保存派生关系。删除源记忆时：
  - 只由该来源支持的派生记忆会一起清除；
  - 仍有其他有效来源的记忆会降低置信度并标记“待复核”；
  - 待复核记忆在用户确认前不能进入召回或模型上下文。
- 与被删除记忆关联的旧对话事件会替换为隐私占位符；统一心智事件也会清除原文并重新回放。
- 引用了相关记忆的每日手记会失效；再次打开时只根据仍然有效的记忆重新生成。
- 导出只包含仍然有效且允许导出的记忆，不包含已删除原文。

### 来源权限

- Level 0 用户主动输入与经过验证的本地推导默认开启。
- Level 1 手动导入和 Level 2 应用状态默认关闭。
- 来源关闭后立即退出召回与模型上下文，但仍可在本地管理页查看并决定是否删除。
- “撤回并删除历史”会同时执行来源关闭和级联清理。
- 未登记的新来源默认关闭，不能因为适配器新增而自动获得模型上下文权限。

### 个人房间

桌宠右键菜单新增“打开她的房间…”，打开 420×560 纵向窗口。房间与桌宠、模拟器
共用同一个 `MindEngine` 和 SQLite 数据库，但只读取严格的公开投影。

- 心情：一个公开百分比和自然语言标签，不展示内部原因、精力或压力。
- 好感度：只显示面向用户的总体亲近程度，不展示信任、安全、尊重等关系分项。
- 重要日记：只显示重要性达到公开门槛的少量日记；普通日常和内部反思保持不公开。
- 一键锁定：立即在桌面界面遮挡房间内容；这是本地视觉遮挡，不冒充 Windows 账户加密。

任务操作、AI 设置和“关于我的本地记忆”治理位于独立入口。记忆治理保留查看、纠正、
删除和导出权，是用户控制自身数据的隐私功能，不代表用户可以读取角色的全部记忆或内心。
成长阶段、证据链、关系分项、状态原因和全部记忆不会进入普通个人房间。

离线恢复只根据经过时间恢复精力并生成可解释摘要，不虚构屏幕外活动或社会经历。

## 闸门验证

- 未授权来源不能写入可用记忆，也不能进入模型上下文。
- 删除测试覆盖原记忆、检索索引、旧事件、统一心智事件、单来源派生、多来源降级和手记重建。
- 阶段 1 旧数据库会原地迁移，已有记忆继续可读，不要求用户清库。
- Windows 真实界面检查确认普通房间只包含心情、总体好感、重要日记与视觉锁定，不暴露内部字段。

阶段 2 的功能闸门已关闭。完整 MVP 仍需阶段 3 的结构化社会理解与提示注入防护，
以及发布前实际跑满阶段 1 的 8 小时桌面稳定性报告。
