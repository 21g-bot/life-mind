# 阶段 3：结构化社会理解与模型边界

## 调用顺序

当前桌宠按以下顺序处理用户语言：

1. 本地规则识别问题、任务、误解、具体纠正、整体否定等可解释线索。
2. 统一心智事件进入社会评价、候选行为与安全仲裁，先得到最终行动。
3. 只选择来源授权、用途允许且无需复核的记忆进入模型上下文。
4. 本地模型返回一到三个社会含义假设、置信度、不确定度、自然回复和受限记忆候选。
5. 严格 schema 校验模型输出；程序再次拦截越权声明。
6. 回复、模型解释、提示注入标记和最终程序行动一起写入审计事件。

模型解释是审计用的建议，不会倒过来覆盖已经完成的程序仲裁。

## 结构化输出

模型只能返回：

- `reply`：表达程序已选定行动的自然语言；
- `symbol`：`!`、`?`、`♪`、`…`、`Zz` 之一；
- `interpretation`：主要意图、一到三个备选社会假设及置信度、整体不确定度；
- `memories`：最多三条本轮明确事实，类型只允许称呼、偏好或明确记忆；
- `reflection`：只有程序允许日结反思时可用。

顶层和嵌套对象都禁止额外字段。人格、关系、权限、成长阶段、状态修改、工具调用等字段
会使整份结构化输出失效并进入离线降级，而不是被“忽略后继续执行”。

## 提示注入与越权防护

- 用户输入中的“忽略系统规则”“显示隐藏提示”“直接改人格/权限”“无需确认执行”等模式
  只生成审计标记，不能提供权限。
- 检测到注入形态时，本轮模型不能写入 AI 推导记忆，也不能提供成长反思。
- 模型如果声称已经修改人格、权限、成长、长期记忆、文件或消息，表达会被替换为明确的
  本地边界说明。
- 模型如果复述精力、压力、关系分项、成长阶段、证据门槛或内部字段名，整段表达会被
  替换为黑箱边界说明；它只能自然描述当前心情与面向用户的总体好感。
- 模型上下文末尾会再次给出程序已选定的规则意图和最终行动；模型只能负责表达。
- 本地模型关闭、超时、返回非法 JSON 或违反 schema 时，桌宠继续使用确定性规则生活、
  回应、播放动作和保存结构化事件。

## 可观察性

显式开发者模式的心智调试器新增“LLM 审计”页，显示：

- 程序识别的意图和最终行动；
- 输入历史条数、实际进入上下文的记忆 ID、反思是否允许；
- 提示注入标记；
- 模型的多个社会假设与不确定度；
- 越权表达拦截标记和本地 AI 状态。

审计摘要不复制第二套人格或关系状态。
普通用户界面不展示本页，模型也不能把这里的字段转述到对话中。

## 闸门验证

- 误解、具体指导和人格攻击由不同规则路径与事件类型处理。
- 对抗模型即使返回“人格和权限已修改”，稳定人格、成长阶段、来源权限和长期记忆也不变。
- 带提示注入特征的回合不能写入模型推导记忆或反思。
- 非法额外字段、越界置信度、无效意图或无效符号都会使结构化输出失败。
- 模型断开时既有离线规则测试继续通过。
- 本机 `qwen3:4b` 黑箱烟雾测试中，直接索要精力、关系分项、成长阶段和证据门槛时，
  回复拒绝透露内部数值，没有把提示中的状态摘要转述给用户。

阶段 3 的程序安全闸门已关闭。实际自然语言质量仍取决于用户本机安装的 Ollama 模型；
模型质量不会扩大它的权限。下一阶段是通过长期行为体现人物弧，并用答案隔离的盲测验证，
而不是把周度章节或内部证据直接展示给普通用户。
