# 阶段 4：行为人物弧、内部证据与盲测

## 目标

这一阶段不再增加一套“成长数值”，而是让阶段 0 已验证的结构化证据改变长期行为。
角色不会用一句“我成长了”证明变化，普通用户也不能查看阶段、证据门槛、潜质清单或
完整关系过程。研究工具生成的纪念物与章节只用于开发验收和答案隔离的行为盲测。

## 证据流

```text
已授权事件 -> 确定性心智仲裁 -> 可回放轨迹
          -> 可观察行为信号 -> 开发验收投影 / 盲测材料
```

- `life_mind/growth_visibility.py` 是纯程序开发验收派生层，不推进阶段，也不调用语言模型。
- 每个信号、纪念物和潜质线索都保存一个或多个 `evidence_event_ids`。
- 同一数据库重启回放后会得到完全相同的纪念物。
- 已删除或已脱敏的私人事件不能继续解锁可见物品或叙事。
- 周章节只描述最近七天发生过的行动；没有证据时明确保持普通日常，不制造转折。

## 可见人物弧

首条人物弧现在有以下用户可观察片段：

1. 连续任务总被直接接下，工作之外尚未留下安排。
2. 她开始注意到疲劳。
3. 疲劳导致具体失误后先暂停复盘，不用更多工作惩罚自己。
4. 面对夹带羞辱的批评，承认具体错误并保留边界。
5. 低紧急度任务到来时先协商范围。
6. 在不同情境中选择私人画画、允许失败的小练习和主动休息。
7. 结构化反思只在前述行动证据满足门槛后参与整合。

## 开发验收纪念物

| 纪念物 | 解锁依据 |
|---|---|
| 第一张只为自己画的速写 | 一次带独立选择证据的私人画画 |
| 留着重试痕迹的练习卡 | 一次允许失败和重试的小技能练习 |
| 十分钟休息便签 | 先协商低紧急度任务，再实际完成主动休息 |
| 重新系好的丝带 | 承认、责任、补救、行为改变、时间证据全部出现 |
| 窗边的普通日子小册 | 至少三次、两个情境、真实代价和证据一致反思齐备 |

这些纪念物只出现在开发预览、自动化验证和盲测材料中。普通个人房间没有“回忆架”；
用户通过行为变化和偶尔公开的重要日记感受人物弧，而不是直接检查证据清单。

## 开发验收潜质线索

潜质只作为开发验收线索，不直接升级成固定人格或天赋，也不进入普通用户界面：

- 两次私人画画：可能愿意把创作留在普通生活里。
- 一次允许失败的练习：可能愿意做一个初学者，仍需更多情境验证。
- 批评边界与任务协商同时出现：可能形成温和而具体的边界能力。

## 行为盲测

生成一份不含内部阶段名、门槛名和答案的参与者材料：

```powershell
py -B tools\run_growth_blind_review.py
```

输出：

- `tmp/growth-blind-review.md`：给体验者填写。
- `tmp/growth-blind-review.json`：结构化参与者材料。
- `tmp/growth-blind-review-answer.json`：单独保存的答案与评分规则，不交给体验者。

给每位体验者复制一份 JSON，填写不同的匿名 `reviewer_id`、两项卡片选择、两条行为证据、
确定度和关键证据，然后只把答卷放入同一目录（例如 `tmp/blind-responses`）。汇总时运行：

```powershell
py -B tools\evaluate_growth_blind_results.py tmp\blind-responses
```

汇总报告默认写入 `tmp/growth-blind-evaluation.json`。重复编号、不完整答卷和版本不一致的
答卷不会计入完成数；少于 5 份完整真人答卷时状态只能是 `pending`。主持人仍需确认体验者
未看过答案，并人工检查两条证据确实来自卡片，程序不会用语言模型替体验者补答案。

建议人工闸门：至少 5 名未看过项目阶段说明的体验者中，4 名能够从具体行为判断哪一组经历表现出更强的任务外生活与边界，并至少指出两条行为证据。自动测试只验证材料不泄露答案、证据完整和回放一致，不能冒充真实体验者结论。

## 当前验证

- 完整测试套件：116 项通过，并在 Python 3.14 与项目运行时 Python 3.12 分别回归。
- 参考人物弧：跨工作、失败、关系、私人时间、学习与休息等情境产生可见信号。
- 100 个低强度模拟日：不会误解锁最终小册。
- 关系修复：四种证据不会解锁丝带，第五种时间证据出现后才解锁。
- 私人事件脱敏：对应私人画作不再显示。
- 数据库重启：可见物品与章节一致。

尚未完成的发布闸门：真实用户盲测，以及阶段 1 的完整 8 小时桌面稳定性运行。
