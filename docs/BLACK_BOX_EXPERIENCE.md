# 黑箱体验与公开信息边界

## 产品原则

桌宠的心智引擎可以拥有复杂的内部状态，但普通用户不应得到一张实时“脑内仪表盘”。
用户需要通过她持续的动作、节奏、选择、语气和少量自愿公开内容逐渐理解她。

这不是用神秘感掩盖随机程序。内部变化仍然必须有结构化事件、证据门槛、确定性回放和
安全审计，只是这些工程数据不等于角色愿意公开的信息。

## 普通用户可以看到

- 当前心情：公开后的总体值与自然语言标签；
- 总体好感度：面向用户的一个聚合值与标签；
- 重要日记：重要性达到门槛、且已经生成公开版本的少量条目；
- 角色行为：动作、回应节奏、主动活动、边界表达和长期选择。

个人房间的数据接口必须保持白名单：`locked`、`mood`、`affection`、
`important_journal`。增加任何公开字段都需要重新做产品边界审查。

## 普通用户不能看到

- 精力、疲劳、压力、需要和注意预算；
- 心情的内部原因与事件解释链；
- 信任、善意、安全、尊重、修复信心等关系分项；
- 成长阶段、冲突名、证据门槛、缺失证据和候选动作得分；
- 全部回忆、全部日记、反思草稿、模型提示词或隐藏推理；
- 调试事件、回放轨迹和 LLM 审计。

模型回复也受同一边界约束。即使本地模型看到了用于控制语气的内部摘要，也不得复述
原始数值、字段名、阶段或证据门槛；检测到这类内容时整段回复会被边界说明替换。

## 用户数据治理不是读心

用户仍然有权管理“关于我的本地记忆”：查看来源、纠正错误、删除、撤回授权和导出。
这是隐私与数据主体权利，不代表角色的全部记忆和内心对用户开放。该入口必须与个人房间
分离，并在界面中明确说明用途。

## 开发者模式

开发、自动化验收和问题诊断可显式运行：

```powershell
py -B run_pet.py --developer-mode
```

此模式才显示状态摘要、心智调试器、关系分项、成长证据与 LLM 审计。普通启动默认关闭，
不能因为数据库已有调试数据或模型返回特殊内容而自行开启。

## 验收要求

1. 普通个人房间的序列化结果只包含公开白名单字段。
2. 普通右键菜单没有状态摘要和心智调试器。
3. 模型复述隐变量时回复被拦截。
4. 重要日记不包含精力百分比、动作内部名、成长阶段或冲突字段。
5. 用户数据删除和导出功能继续独立可用。
6. Windows 实机截图不得出现隐藏字段或旧的书桌、回忆架、关系册和权限页。
