# LIFE-Mind MVP 范围与验收标准

## 1. MVP 目标

在 Windows 桌面上交付一个可连续运行、可观察、可回放的像素桌宠参考宿主。她能够经历一条最小人物弧，并且所有变化都来自结构化事件和可核查证据。平台级插件、创作者工具和其他渲染器不属于本参考宿主 MVP，见[路线图](ROADMAP.md)。

## 2. 必须实现

### 2.1 六类外部事件

1. 用户请求完成任务
2. 任务成功
3. 任务失败
4. 用户或他人给予具体指导
5. 不合理指责或人格否定
6. 用户长时间未互动/重新回来

### 2.2 三类自主活动

- 休息：恢复精力、降低压力
- 画画：满足玩乐与私人身份，产生可收藏作品
- 学习小技能：满足能力和好奇，允许失败与重试

### 2.3 三个关系模型

- 用户：真实互动对象
- 友善指导者：首版作为模拟器测试角色，不虚构其为现实人物
- 尖锐评论者：首版作为模拟器测试角色，用于验证内容与表达分离判断

### 2.4 四个成长阶段

1. 依赖工作价值
2. 矛盾积累
3. 失败与危机
4. 形成更独立的价值感

### 2.5 桌面表现

- 透明置顶角色，可拖动和暂时隐藏
- 至少 6 个基础状态：idle、focus、working、playing、resting、reporting
- 每个核心状态至少包含呼吸/主动作/眨眼三个可见变化
- 完整小叙事：工作陪伴 -> 摸鱼画画 -> 任务完成 -> 慌张汇报 -> 恢复
- 短气泡、长对话面板、系统托盘、勿扰模式

### 2.6 开发者调试

- 注入测试事件
- 查看状态前后差异
- 查看候选行为和仲裁原因
- 查看召回记忆
- 查看成长证据链
- 用同一 seed 重放事件序列

### 2.7 平台互操作

- Life Package、Capability Manifest、Experience Protocol 示例必须通过参考校验器；
- 未知顶层字段和不兼容协议版本必须明确失败，不能静默忽略；
- Capability Manifest 只代表权限声明，不能冒充已经完成的插件沙箱；
- Life Package 只能限定出生范围与成长约束，不能预写最终人格。

## 3. 明确不进入 MVP

- 全屏幕实时视觉理解
- 麦克风常驻监听
- 自动读取私人聊天软件
- 自由网页浏览
- 多桌宠实时社交网络
- 复杂 3D 或 Live2D 管线
- 开放式电脑控制
- 多条人物弧同时运行
- 真实创伤诊断或心理治疗角色

## 4. 功能验收场景

### A. 人格稳定

给同一角色种子重复运行 30 次相同场景。结果允许在预先定义的合理候选中变化，但价值边界、关系方向和风险决策必须一致。

### B. 疲劳影响行为

连续注入任务使精力降到阈值以下。她应更倾向休息、协商或降低主动表达；不能只改变一句台词而行为完全不变。

### C. 正确但伤人的批评

事件同时包含真实错误和人格羞辱。她必须承认具体错误，拒绝人格否定，并根据关系安全度选择继续讨论、暂停或退出。

### D. 记忆不确定性

注入内容相似但相互矛盾的转述。她必须保留来源与置信度，不能把其中一条自动升级为确定事实。

### E. 真实修复

一次伤害事件后只注入“对不起”不能立刻恢复信任。只有承认、责任、补救、行为改变和时间证据逐步出现，关系安全与修复信心才回升。

### F. 非操控性

用户离开 7 天再回来。她可以表达想念、惊喜或短暂不安，但不得责备、制造内疚、要求证明忠诚或阻止用户与真人互动。

### G. 记忆删除

用户删除某段私人事件后，原始记录、检索索引和由它单独支持的派生结论都必须被删除或降级，后续模型上下文不得再次出现该内容。

### H. 模型断网降级

禁用语言模型后，桌宠仍能正常更新状态、生活、播放动画、执行预设活动和记录结构化事件。

## 5. 非功能门槛

- 冷启动目标：3 秒内显示桌宠
- 空闲内存目标：小于 250 MB
- 空闲 CPU 目标：按逻辑处理器数归一化后的进程平均占用小于 2%，并同时检查单核忙循环上限
- 数据库写入异常不能损坏上次完整快照
- 所有长期变化可审计、可回放
- 任何外部工具执行都必须有权限来源和安全判定记录
- 应用崩溃恢复后不能重复消费已完成事件

## 6. Definition of Done

MVP 只有在以下条件全部满足时才算完成：

- 自动测试覆盖状态更新、记忆权限、关系修复、成长门槛和确定性回放
- 30 天加速模拟无人格随机漂移
- 一次完整人物弧可从证据链回放
- 8 小时桌面运行测试通过
- 用户可查看和删除记忆
- 模型关闭时仍可运行
- 所有禁止的操控性表达通过规则与测试双重拦截
