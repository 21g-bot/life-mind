# 数据可靠性与恢复

LIFE-Mind 的桌宠数据默认只保存在 `%LOCALAPPDATA%\LIFE-Mind\`。本页说明程序如何保护这些
长期数据，以及使用者和维护者可以执行哪些不会泄露私人心智内容的操作。

## 启动顺序

1. 以只读方式执行 SQLite `quick_check`、外键检查、应用标识和架构版本检查；
2. 旧版无版本数据库在迁移前创建一致性快照；
3. 在单一事务中创建或补齐当前架构，最后写入 `user_version` 和迁移记录；
4. 更高版本数据库会被拒绝，不会被旧程序降级或隔离；
5. 物理损坏或外键不一致的数据库会连同 `-wal`、`-shm` 移入 `recovery\`；
6. 程序从新到旧检查备份，跳过损坏快照并恢复最近的有效副本；没有有效备份时才新建数据库。

隔离表示移动到用户自己的恢复目录，不表示删除。自动恢复后桌宠会显示一次明确提示，但黑箱界面
不会展示记忆正文、内部关系分项或本机路径。

## 自动与手动备份

默认用户数据库在正常退出时通过 SQLite Backup API 生成一致性快照。快照先写入临时文件，
通过 `quick_check` 后才原子替换为正式 `.db` 文件；默认只保留最近 7 份。AI 密钥由操作系统
凭据库保存，不在 SQLite 数据库或备份内。

右键桌宠选择“备份本地数据”，或者在桌宠退出后运行：

```powershell
python -B run_pet.py --backup-now
```

手动恢复最近有效备份：

```powershell
python -B run_pet.py --restore-latest-backup
```

恢复前，当前数据库和 WAL 边车会先进入 `recovery\manual-restore-*\`。命令只返回备份文件名和
是否完成隔离，不输出私人绝对路径。不要在另一个 LIFE-Mind 进程仍持有数据库时执行恢复。

## 脱敏诊断

```powershell
python -B run_pet.py --doctor
```

Windows 便携版可把 `python -B run_pet.py` 换成 `.\LIFE-Mind.exe`；窗口版没有控制台时会用
系统对话框显示同一结果。

退出码为 `0` 表示数据库与动作库可正常使用，包括“尚未首次创建”和“可迁移旧架构”；退出码
为 `1` 表示需要处理。报告只包含：

- LIFE-Mind、Python 和架构版本；
- 数据库大小、表数、事件数、活动记忆数、外键错误数、无效 JSON 字段数和缺失表数；
- 有效备份数量；
- 动作库风格、动作数和总帧数。

报告不包含用户名、本机路径、对话、记忆正文、日记、模型地址、模型回复或 API 密钥，可作为
GitHub Issue 的诊断附件。仍应在发布前人工浏览一次内容。

## 长期数据验收

维护者可以运行可重复的数据可靠性闸门：

```powershell
python -B tools\benchmark_data_reliability.py --events 10000
```

它会在临时目录生成 10,000 条确定性事件，关闭并重新打开数据库，创建经过完整性检查的备份，
主动破坏工作数据库，再验证自动恢复后的事件数和状态摘要完全一致。默认预算为：写入 120 秒、
启动 20 秒、备份 20 秒、恢复 30 秒、数据库 256 MiB、进程 RSS 512 MiB。报告原子写入
`tmp/data-reliability-benchmark.json`，测试数据库随后删除，不接触用户数据。

2026-07-19 的 Windows x64 参考验收结果：10,000 条事件写入 33.86 秒，重启重放 6.58 秒，
备份 1.12 秒，损坏后自动恢复 7.42 秒；数据库约 78.7 MiB，峰值 RSS 约 51.2 MiB，全部闸门
通过。硬件不同会影响耗时，因此自动判定以预算为准，参考数字不作为跨设备性能承诺。

完整审计轨迹仍保存在 SQLite 中。常驻运行时只缓存最近 128 条完整模拟轨迹；读取更早的重复
事件时会从数据库恢复原轨迹，因此不会改变幂等性，也不会随历史长度无限占用内存。

## 维护边界

- 备份、恢复目录和数据库都属于私人数据，不进入公开仓库或 Release；
- 旧程序绝不自动打开或修改更高 `user_version` 的数据库；
- 迁移失败会回滚，迁移前快照保持可再次升级；
- 自动恢复只处理可判定的数据损坏，不会把“不属于 LIFE-Mind”的 SQLite 文件当作桌宠数据库；
- 当前实现不能替代操作系统级整盘备份；强制终止和未提交 WAL 回滚已有自动测试，但仍需在
  发布候选版上执行真实断电演练。
