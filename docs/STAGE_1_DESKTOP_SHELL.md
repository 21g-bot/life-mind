# 阶段 1：桌面外壳与稳定性

## 已实现

- Windows 系统托盘：显示/隐藏桌宠、切换勿扰、暂停动画和退出。
- 隐藏保护：托盘未成功启动时拒绝隐藏，避免桌宠消失后无法恢复。
- 专注/勿扰模式：停止未经请求的自主活动与气泡，保留眨眼、用户主动对话和手动操作。
- 配置持久化：勿扰状态与位置、大小、置顶和待机动画一起保存在本地配置。
- 正式像素动作按需加载：启动时先显示 `idle`，需要某个动作时才加载；回到稳定动作后释放不用的 Tk 帧缓存。
- 快速首帧：窗口先映射第一张正式像素帧，再建立完整待机缓存，避免加载动画时桌面长时间没有反馈。

系统托盘依赖 `pystray==0.19.5`，项目启动器会优先读取 `.deps/python`。依赖可用以下方式安装：

```powershell
py -m pip install --target .deps\python -r requirements.txt
```

## 稳定性门槛

`life_mind.soak` 使用 Windows 原生进程计数器采样，不依赖 psutil。记录：

- 当前与峰值工作集；
- 5 分钟预热后的稳定期最大值与 P95 工作集；
- 稳定期首尾窗口的低四分位工作集基线，以及至少运行 1 小时后才启用的每小时增长率；
- 句柄的窗口基线增长及每小时增长率；
- CPU 累计时间与单核平均占用；
- 每个采样点和最终闸门结果。

Windows 的峰值工作集是“进程启动以来的累计最高值”，会包含监控开始前的启动和首批贴图
解码。因此它只作为 512 MiB 的失控硬上限，不能单独代表 8 小时稳态。正常运行主要由稳定期
最大值 384 MiB、P95 不高于 320 MiB、基线增长不高于 64 MiB、长期增长不高于每小时 8 MiB
共同约束。这样既不会放过持续泄漏，也不会因为一次已经释放的启动峰值误判。

一键启动隔离桌宠并运行 8 小时：

```powershell
py -B tools\run_desktop_soak.py --hours 8 --sample-seconds 60
```

Windows 监控线程会在采样期间请求系统保持唤醒，但允许屏幕按原计划关闭；测试结束或异常退出时
立即撤销请求，不会永久修改当前电源计划。合盖、关机、重启和系统强制睡眠仍会中断测试。

默认报告写入 `tmp/desktop-soak-report.json`。运行期间每次采样都会原子更新
`tmp/desktop-soak-status.json`，其中包含状态、桌宠 PID、已运行秒数、采样数和最新资源值；
因此长时命令即使没有持续打印终端输出，也可以区分“仍在运行”和“已经中断”。原始采样还会
逐次原子写入同目录的 `.partial.json` 检查点；正常完成后自动删除，异常中断时保留，避免数小时
样本全部丢失。报告还会核对目标时长完成率；中断检查点可以重评资源趋势，但不会被标成完成了
8 小时正式闸门。产品的空闲目标仍是低于 250 MiB；较高的稳定期上限用于允许坐立过渡短暂
同时持有离场、过渡和入场动作。

旧策略产生的完整报告或异常保留的检查点，可以用当前策略重评：

```powershell
py -B tools\reassess_soak_report.py tmp\desktop-soak-report.json
```

重评会生成独立的 `*-reassessed.json`，保留旧报告，不会篡改原始测试证据。

## 当前验收记录

- 当前完整项目 173 项自动化测试通过，固定人物弧重复回放 100 次结果一致。
- Windows 真实界面确认只显示当前动作库素材；绘画与返回站姿过渡没有横向漂移或缩放跳变。
- 勿扰模式真实开启后会回到安静待机，5 秒观察期内没有再次触发自主动作。
- 勿扰待机实测工作集约 125.5 MiB、349 个句柄；加载绘画动作时当前工作集约 194.8 MiB。
- 72 秒自动短测采样 25 次，峰值约 291.6 MiB，稳态工作集基线增长约 0.2 MiB、句柄增长 2，
  报告保存在
  `tmp/desktop-soak-smoke.json`。这只是入口与采样链路的冒烟验证，不能替代 8 小时闸门。
- 策略 v2 冷启动实测 30 秒、8 个采样点：累计峰值约 306.4 MiB、当前工作集采样最大约
  276.2 MiB、句柄减少 4、单核平均 CPU 约 8.7%；目标时长、资源闸门、逐次检查点清理及独立
  子进程日志均验证通过。短测仍不能替代 8 小时闸门。
