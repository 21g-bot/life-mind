# Changelog

本项目使用语义化版本号。正式发布前的重大接口仍可能调整。

## [Unreleased]

## [0.1.2] - 2026-07-18

- 心智事件改为增量追加，避免长期使用时随历史增长产生二次方级回放开销；写入失败仍会回滚到最后一次已提交状态。
- 限制异常用户输入和模型 HTTP 响应体大小，并严格拒绝类型不符合公开结构化契约的模型输出。
- 检测到提示注入的消息不再进入长期记忆，记忆上下文明确作为数据而非可执行指令。
- 模型请求的 `User-Agent` 现在携带真实 LIFE-Mind 版本号。

## [0.1.1] - 2026-07-18

- 修复桌宠配置文件为非对象 JSON 时的启动崩溃，并严格解析布尔值、缩放和坐标类型；
- 远程与局域网 AI 接口强制使用 HTTPS，本机回环服务仍可使用 HTTP；
- 凭据绑定保留 URL 路径大小写，避免大小写敏感的多租户接口共用错误密钥；
- 慢模型请求期间暂停主线程的自主心智刷新，并在退出后异步等待请求收尾，避免窗口卡住；
- 将 Windows 产物上传与下载工作流迁移到 Node.js 24 版本的官方 Artifact Actions。

## [0.1.0] - 2026-07-17

- 建立公开框架与私人角色数据隔离边界；
- 增加 Apache License 2.0、发布检查和 GitHub 自动测试；
- 增加中立演示角色、角色模板与中文快速入门；
- 增加本地 AI、黑箱个人房间、记忆治理和成长证据的公开实现。
- 将项目重新定位为可嵌入不同表现层的开放式角色生命引擎；
- 增加 Life Package、Capability Manifest、Experience Protocol 0.1 的运行时校验、
  JSON Schema、公开示例与自动化测试。
- 增加模型、语音、能力和渲染器扩展端口，以及黑箱安全的 `PresentationIntent` 表现压缩层；
- 增加七个参考项目的官方资料吸收矩阵、带依赖与验收条件的 GitHub 实施任务和任务模板。
- 将单一 Ollama 接入扩展为 Ollama、OpenAI 兼容和 Anthropic 三类适配器，增加常用厂商预设、
  系统凭据库、云端数据许可、长期记忆共享开关和基于当前消息证据的模型记忆准入。
- 增加 PyInstaller Windows x64 便携构建、冻结版隔离运行检查、SHA256 校验和标签驱动的
  GitHub Release 工作流；冻结版不会打入私人角色，也不会向程序内部目录写用户数据。
